6. Matheuristics

e Matheuristic is a change of paradigm about the we work with MIP model. We need to think the MIP as a black box, and leave the current state of the art implementation as is. Using this rule, the only way we can operate, is from outside, especially inside the MIP model. Matheuristic suggest some heuristic constraints (from the name math + heuristic) trying to help the model to find good solution.

This paradigm have a lot of advantages about: first of all we don't need to develop every time an "on-purpose" MIP solver, spending our time work on our specific problem, secondly, using commercial node, we can count in a top performance MIP solver, with a lot of effort and optimization. In this section, we're talking about two type of matheuristic: diving and local branching.

Diving

Diving consists in fix some variable in order to help the model to find a solution faster, since the problem have less variables and execute it many times in order to do tiny improvement to solution. In this specific scenario, we're going to fix a number of edges suggesting to the model that this edges are included into the solution. In our implementation, the degree of freedom increase dynamically when model not improve too much and decrease otherwise, from 10% to 60% of free arc into a solution. After solving the model, we need to remove the second constraint for it e turn back to (1.5) if the new solution provided by model is better than before, we can use that solution as warm start.

Which Policy use to Fix Variables

A good idea to solve the TSP quicker with diving is to block only the edge we're sure is in the solution, and change the others. Since define certainly which edge is kept into solution is impossible, we can greedily approach the problem. We use two approaches:

  1. Random: pick randomly edges to block, this is simple to implement and work without smart assumption.
  2. Weighted: pick edges with cost greater or equal to average cost of the current solution. This implementation came from assumption that, edge with greater cost than average, is unlike to be in the final solution (remember that is a greedy approach, we don't have any proof).

In our specific implementation, in order to make the weighted procedure less deterministic, we use to make random procedure 30% of the time.
